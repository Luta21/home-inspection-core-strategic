Service Page Structure — Which layout approach?

Each of your 6 services (Verificare documentație, Verificare structuri, Scanare termografică, Verificare instalații, Umiditate structuri, Expertize tehnice) needs its own page. How should these pages be structured?

A

Immersive Long-Scroll (Recommended)

Each service page is a full conversion funnel — hero with service-specific imagery, problem/pain section, detailed methodology, equipment used, deliverables, mini pricing CTA, FAQ, and final CTA. Similar to homepage flow but focused on one service. Best for SEO and conversions — each page ranks independently and guides the visitor to action.

Pros

  • Maximum SEO juice per service keyword
  • Full conversion funnel per page
  • Tells a complete story
  • Matches your premium dark+gold aesthetic

Cons

  • More content to create per page
  • 6 pages × 8-10 sections each
B

Modular Template with Service-Specific Data

One shared layout template with sections that adapt based on service data (constants). Hero, features grid, process, equipment, CTA — same skeleton, different content. Faster to build, consistent feel.

Pros

  • Build once, reuse 6 times
  • Consistent UX across services
  • Easier maintenance

Cons

  • Less personality per page
  • Risk of feeling generic
  • Some services may not fit the template well
C

Hybrid — Shared Template + Unique Hero & Key Section

A reusable template for the common sections (methodology, equipment, FAQ, CTA), but each page gets a unique hero treatment and one signature section that differentiates it (e.g., thermography page gets an interactive before/after, structural gets a diagram). Best of both worlds.

Pros

  • Efficient to build
  • Each page still feels unique
  • Strong SEO with unique content
  • Scalable for future services

Cons

  • Need to design 6 unique hero/signature sections
  • Slightly more complex component architecture
